IIQ Tasks

Here's a breakdown of IdentityIQ tasks, their purposes, and suggested scheduling intervals, based on the provided sources:

Tasks and Their Purposes

  • Access History Task: This task extracts IdentityIQ objects (identities, roles, accounts, etc.) that are new or have changed, formats them into JSON, and enqueues them for the Access History writer service. This is used to populate the access history store.
  • Account Aggregation Task: This task scans applications to discover users and entitlements, then correlates that information with roles in IdentityIQ. This task can also create new identities if accounts are discovered that do not have matching identities.
  • Account Group Aggregation Task: This task scans applications to aggregate information about groups, which can be used for group certifications or for displaying group information in identity certifications.
  • Activity Aggregation: This task is used to scan selected applications for activity targets.
  • Effective Access Indexing Task: This task is used to track classification data for effective access items.
  • Encrypted Data Synchronization Task: This task is used to re-encrypt IdentityIQ data when a new custom encryption key is generated.
  • File Access Manager Classification Task: This task is used to integrate with File Access Manager to discover and govern access to sensitive data.
  • Identity Refresh Task: This task scans all identities to ensure that all identity information is up-to-date and accurate. It is also used to detect and report policy violations. Identity refresh can be configured to only refresh identities that have changed since the last refresh, using a "delta" refresh.
  • Identity Request Maintenance Task: This task scans Lifecycle Manager access requests to ensure that all change requests were provisioned. It also prunes old identity request objects.
  • ITIM Application Creator Task: This task inspects IBM Tivoli Identity Manager (ITIM) and retrieves information about the ITIM services (applications).
  • Missing Managed Entitlements Scan Task: This task scans a selected application to create entitlement objects for items added after the application was last aggregated.
  • OIM Application Creator Task: This task inspects the OIM application and retrieves information about all connected applications.
  • Policy Scan Task: This task runs policies against identity cubes and updates identity score cards with any policy violations discovered.
  • Propagate Role Changes Task: This task refreshes identities that have an assigned role whose associated entitlements have changed. It can also be configured to run policy checking.
  • Refresh Logical Accounts Task: This task is used to refresh composite accounts for all identities that could potentially have a composite account on the composite applications selected.
  • Role Index Refresh Task: This task updates all role information and creates the indexes needed to perform role searches. This task must be run before performing any role searching.
  • Role-Entitlement Associations Task: This task deletes existing role-entitlement associations, then analyzes each role in the system and creates associations between the role and any granted entitlements.
  • Run Rule Task: This task runs a specified rule with name/value pairs.
  • Sequential Task Launcher: This task lets you run a series of tasks sequentially, without having to schedule them individually.
  • System Maintenance Task: This task performs a variety of background maintenance activities, such as pruning identity snapshots, task results, and certifications, and escalating orphaned work items. It also includes the options for Check Expired Mitigations, Check Expired Work Items, and Check Sunset Requests.
  • Target Aggregation Task: This task scans selected applications for activity targets.

Suggested Scheduling Intervals

The scheduling frequency of tasks depends on the specific needs of an organization and the nature of the data being processed. Here's a general guide based on the provided sources:

  • Access History Task: The default configuration is to run daily at midnight. However, it can be scheduled to run at a different time or cadence as needed.
  • Account Aggregation Task: The schedule depends on the importance of the source application.
    • For authoritative applications, this may be hourly.
    • For other applications, it may be daily or weekly.
  • Account Group Aggregation Task: The schedule of this task is not explicitly mentioned in the sources.
  • Effective Access Indexing Task: The sources do not specify how often this task should be run, but it should be run on a recurring basis.
  • File Access Manager Classification Task: This task should be run on a recurring basis.
  • Identity Refresh Task:
    • The schedule of this task depends on aggregation schedules and data calculation needs.
    • Can be run daily or weekly.
    • A delta refresh can be run hourly with a full refresh daily.
    • Should be run after aggregation tasks.
    • The Identity Refresh task that launches workflow triggers should be configured as a full refresh.
  • Identity Request Maintenance Task: This task is pre-scheduled to run once a day by default, but it could be run more often as needed.
  • ITIM Application Creator Task: The sources do not specify how often this task should be run.
  • Missing Managed Entitlements Scan Task: The sources do not specify how often this task should be run.
  • OIM Application Creator Task: The sources do not specify how often this task should be run.
  • Policy Scan Task: The sources do not specify how often this task should be run.
  • Propagate Role Changes Task: The schedule of this task depends on the volume of role model changes.
  • Refresh Logical Accounts Task: The sources do not specify how often this task should be run.
  • Role Index Refresh Task: This task should be run before performing any role searching. The sources do not specify the frequency of this task.
  • Role-Entitlement Associations Task: The sources do not specify how often this task should be run.
  • Run Rule Task: This task can be scheduled hourly or more frequently by using the debug pages.
  • Sequential Task Launcher: The schedule of this task depends on the frequency of the tasks it includes.
  • System Maintenance Task:
    • The Perform Maintenance task is pre-configured to run every 5 minutes.
    • Other tasks included in System Maintenance like Check Expired Mitigations, Check Expired Work Items, and Check Sunset Requests are pre-configured to run daily.

Additional Notes

  • Delta Refresh: Using a delta refresh (refreshing only changed identities) can significantly reduce refresh time.
  • Partitioning: For tasks like account aggregation and identity refresh, partitioning can be used to speed up processing time by breaking operations into multiple pieces.
  • Full Refresh: It is recommended to run a full refresh regularly, even when using delta refreshes, to ensure all identities are updated.
  • Task Monitoring: Task administrators should monitor tasks and troubleshoot issues using the Administrator Console.
  • Email Notifications: Email notifications can be configured to be sent upon task completion.
  • Task Results: Task results can be viewed on the Task Results page.

This information is based on the provided sources. Scheduling should be tailored to the specific needs and constraints of your IdentityIQ environment.
