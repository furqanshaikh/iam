Retrieving Certification Actions Taken by Managers

When working with identity governance systems like Oracle Identity Manager (OIM), generating reports that detail certification actions is a critical part of ensuring compliance. In this post, we'll demonstrate how to retrieve information about certifications handled by managers, enriched with data from the CERT_DEFN table.

Scenario Overview​

Our goal is to pull records for certifications processed in the last X days, where managers have taken some action on accounts or entitlements. We want to display not only the details of the certification and the actions taken but also information about the corresponding certification definition, as stored in the CERT_DEFN table.

Key Data Points:​

  • Certification Details: ID, Certification Name, Creation Date.
  • Manager Details: Username, Email.
  • Action Details: Last decision made, Sign-off date.
  • User Details: First name, Last name, Manager, Employee type.
  • Account Details: Entitlement or account-related attribute value and glossary definition.
  • Certification Definition: The name of the certification definition, which allows for grouping and analysis of related certifications.

SQL Query Explanation​

To achieve this, we join the following tables:

  1. cert_certs (CC): This table holds the core certification details, including the certificate name and ID.
  2. CERT_LAST_DECISION (CLD): Stores the most recent actions taken by managers.
  3. CERTS_USER (CDU): Contains user and manager details like usernames and email addresses.
  4. CERTS_ACCOUNT (CA): Holds account-related information.
  5. CERTS_ACCT_ENT_ASGN (CAEA): Stores additional account attributes, such as entitlement assignments.
  6. CERT_DEFN (CD): We join this table to map each certification to its definition, using a case-insensitive LIKE condition to match certificate names.

The updated query looks like this:

SELECT DISTINCT
    cc.ID AS ID, 
    cc.CREATEDATE, 
    cc.CERT_NAME AS CERT_NAME, 
    CDU.USERNAME AS MANAGER_USERNAME, 
    CDU.PRIMARYEMAIL AS MANAGER_EMAIL, 
    CLD.LAST_DECISION AS LAST_DECISION, 
    CLD.SIGN_OFF_DATE AS SIGN_OFF_DATE, 
    CDU.USERNAME AS USERNAME, 
    CDU.FIRSTNAME AS FIRSTNAME, 
    CDU.LASTNAME AS LASTNAME, 
    CDU.MIDDLENAME AS MIDDLENAME, 
    CDU.PRIMARYEMAIL AS PRIMARYEMAIL, 
    CDU.EMPLOYEETYPE AS EMPLOYEETYPE, 
    CDU.MANAGER AS MANAGER, 
    CAEA.ATTRIBUTE_VALUE AS ATTRIBUTE_VALUE, 
    CAEA.GLOSSARY_DEF AS GLOSSARY_DEF,
    cd.cert_name AS CERT_DEFN_NAME 
FROM
    cert_certs cc
LEFT OUTER JOIN CERT_LAST_DECISION CLD
    ON cc.ID = CLD.CERT_ID
LEFT OUTER JOIN CERTS_USER CDU
    ON CDU.IAM_USER_ID = CLD.IAM_USER_ID
LEFT OUTER JOIN CERTS_ACCOUNT CA
    ON CLD.IAM_ACCOUNT_ID = CA.IAM_ACCOUNT_ID
LEFT OUTER JOIN CERTS_ACCT_ENT_ASGN CAEA
    ON CLD.IAM_ATTR_VAL_ID = CAEA.IAM_ATTR_VAL_ID
LEFT OUTER JOIN CERT_DEFN cd
    ON UPPER(cc.CERT_NAME) LIKE '%' || UPPER(cd.cert_name) || '%' 
WHERE 
    cc.CREATEDATE > sysdate - 5
    AND ATTRIBUTE_VALUE is not null
ORDER BY CERT_NAME;

Conclusion​

With this query, you can effectively generate detailed reports on certifications processed by managers, while also pulling in the certification definition name for added clarity and analysis. This approach can help compliance teams understand trends in certification handling and ensure that certification processes align with organizational policies.

By joining with the CERT_DEFN table, the query allows you to group certifications by definition, which can help in auditing and monitoring certification campaigns.
